# TTL 连续采样更新说明 20260601

本版本已经删除 ARTIQ 侧的旧发布周期节流逻辑。

## 当前采样方式

`ttl_dataset_bridge.py` 启动后只在初始化时执行一次 TTL 输入配置：

```python
self.ttl.input()
```

之后主循环会持续执行：

```text
读取 gate_time / gate_subdivisions
在 ARTIQ core 中连续执行 gates_per_batch 个 gate
统计这一批 gate 的 total_count
发布 pulse_latest / counts / ttl_monitor.count_rate / ttl_monitor.sample
立刻进入下一批
```

其中 `gate_subdivisions` 现在表示 `gates_per_batch`，也就是每批连续采样的 gate 数量，不再表示把一个大 gate 切成小段。

## 已删除的内容

以下旧内容已经不再使用：

```text
旧发布周期参数
旧发布周期环境变量
旧 dataset 发布周期配置
旧发布后等待逻辑
```

`03_start_ttl_bridge.bat` 调用 `artiq_run` 时也不再传入旧发布周期参数。

## GUI 刷新

GUI 中的 `GUI刷新(ms)` 只表示窗口多久读取一次 dataset 并刷新图像。它不控制 ARTIQ 采样节奏。

ARTIQ 采样由 `ttl_dataset_bridge.py` 持续进行；GUI 只是读取最新发布的结果。
